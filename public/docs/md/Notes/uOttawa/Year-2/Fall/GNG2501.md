## pourquoi suivre un processus?

### PCTI

1.  Boucle de raffinement du problème

- bien identifie le problem avant de trouver des solution
- s'assurer qu'on resoud le bon problem
  Etape:
  - empathiser avec le client pour identifier ses problèmes
  - Enquête/recherche sur le problem
  - Mantra de conception: "Est ce que je resoud le bon problem"(itératif)

2. Boucle de perseverance

- considere le bon CPX
- apres avoir identifier le bon problem
- Etapes:
  - Critères de conception? CPX considérée
  - générer les solution. Générer le plus de solutions possible
  - prototype et essai
  - Evaluation de la solution. SI oui, terminer. Sinon, retourner a la boucle de perseverance
  - Fondation
    - depend sur votre connaissances de genie, habileté, experience, etc
